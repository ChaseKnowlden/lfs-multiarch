# x86 (IA-32 / i686) Supplement

| Variable         | Value                         |
|------------------|-------------------------------|
| `LFS_TGT`        | `i686-lfs-linux-gnu`          |
| `LFS_ARCH_LINUX` | `x86`                         |
| `LFS_BITS`       | 32                            |
| `LFS_ENDIAN`     | little                        |
| `LFS_ABI`        | IA-32                         |
| `LFS_GCC_EXTRA`  | `--with-arch=i686`            |
| Bootloader       | GRUB BIOS                     |
| Tier             | 1 — well-tested, CI-verified  |

## Platform Notes

- Targets i686 (Pentium Pro and newer) as the minimum instruction set.
- No `lib64` directory needed; all libraries go in `/usr/lib`.
- x86 is a 32-bit build even when the host is amd64 — the host GCC must
  have multilib support (`gcc-multilib` on Debian/Ubuntu, `glibc-devel.i686`
  on Fedora) or use a dedicated i686 cross-compiler.
- **PAE kernel**: Enable `CONFIG_HIGHMEM64G` if building for a system with
  more than 4 GB RAM.  The default 32-bit kernel can only directly address
  4 GB.
- 32-bit x86 is functionally stable but sees less active upstream development.
  Consider amd64 for new builds.

## Building on an amd64 Host

The simplest way to build an x86 LFS on an amd64 host is to use the
cross-compilation path with `LFS_TGT=i686-lfs-linux-gnu` and a cross GCC.
Alternatively, set `CC="gcc -m32"` and `CXX="g++ -m32"` if multilib is
available on the host.
