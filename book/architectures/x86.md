# x86 (IA-32 / i686) Supplement

| Variable         | Value                         |
|------------------|-------------------------------|
| `LFS_TGT`        | `i686-lfs-linux-gnu`          |
| `LFS_ARCH_LINUX` | `x86`                         |
| `LFS_BITS`       | 32                            |
| `LFS_ENDIAN`     | little                        |
| `LFS_ABI`        | IA-32                         |
| `LFS_GCC_EXTRA`  | `--with-arch=i686`            |
| Bootloader       | GRUB BIOS                     |
| Tier             | 1 — well-tested, CI-verified  |

## Toolchain Version Requirements

| Tool     | Minimum version |
|----------|----------------|
| GCC      | 4.8            |
| Binutils | 2.25           |
| Linux    | 4.14           |
| Glibc    | 2.17           |

When building on an amd64 host, the host GCC must also provide 32-bit multilib
support (see [Building on an amd64 Host](#building-on-an-amd64-host) below).

## ISA Baseline

This book targets **i686** (Pentium Pro, 1995) as the minimum instruction set.
The GCC flag `--with-arch=i686` selects this baseline, which includes:

- CMOV (conditional move instructions)
- The P6 branch-prediction and out-of-order execution model

The i686 baseline excludes SSE/SSE2; those are amd64-only assumptions.  If your
hardware supports SSE or SSE2 (Pentium III / Pentium 4 era), you may add
`-msse2` to `CFLAGS` for individual performance-sensitive packages, but the
toolchain itself is built without it for maximum compatibility.

**Why not i386 or i486?**  Glibc 2.26+ dropped i486 support; i686 is the
lowest supported IA-32 baseline in modern Glibc.

## Building on an amd64 Host

Three options are available:

### Option 1: Cross-compiler (recommended)

Set `LFS_TGT=i686-lfs-linux-gnu` and build a dedicated i686 cross-compiler in
the cross-toolchain chapters.  No host multilib packages required.

### Option 2: Multilib host compiler

If the host GCC has multilib support, pass `CC="gcc -m32"` and `CXX="g++ -m32"`
where needed.  Install the 32-bit support libraries first:

- **Debian/Ubuntu**: `apt install gcc-multilib g++-multilib lib32z1-dev`
- **Fedora**: `dnf install glibc-devel.i686 libgcc.i686`

### Option 3: Native i686 host

Building natively on a physical or virtual i686 machine avoids all multilib
complications.  QEMU can emulate i686 with `-machine pc -cpu pentium3`.

## Platform Notes

- Targets i686 (Pentium Pro and newer) as the minimum instruction set.
- No `lib64` directory is needed; all libraries go in `/usr/lib`.
- x86 is a 32-bit build even when the host is amd64.
- **PAE and memory**: The default 32-bit kernel can only directly address 4 GB
  of RAM.  Enable `CONFIG_HIGHMEM64G` (`PAE`) in the kernel configuration to
  support up to 64 GB physical RAM on PAE-capable CPUs (Pentium Pro and later).
  Individual processes are still limited to a 4 GB virtual address space.
- 32-bit x86 is functionally stable but sees reduced upstream development.
  For new builds on modern hardware, amd64 is the preferred target.
- Common test platforms: QEMU (`-machine pc -cpu qemu32`), older physical
  hardware, VirtualBox 32-bit VMs.

## Cross-Compilation Toolchain

### Binutils Pass 1

No architecture-specific flags are required.  The `i686-lfs-linux-gnu` triplet
selects the 32-bit ELF target unambiguously.

### GCC Pass 1

The key flags for x86:

| Flag | Purpose |
|------|---------|
| `--with-arch=i686` | Targets the i686 (Pentium Pro) baseline; set via `$LFS_GCC_EXTRA` |
| `--disable-multilib` | Prevents generation of 64-bit (`-m64`) libraries; only 32-bit is needed |
| `--with-newlib` / `--without-headers` | No C library yet at this stage |

**The `t-linux64` sed is NOT needed for x86.** That sed (`sed -e '/m64=/s/lib64/lib/'
gcc/config/i386/t-linux64`) applies only when building a 64-bit GCC on a 64-bit
host.  When targeting i686, the `t-linux64` file is irrelevant; skip it.

Similarly, **no `lib64` → `lib` symlink** is needed in the sysroot.  All
32-bit libraries go in `/usr/lib` directly.

### Glibc

For i686, Glibc creates the LSB compatibility symlink automatically, but you
must create the library directory first:

```bash
mkdir -pv $LFS/lib
```

The standard Glibc configure is sufficient for i686.  No extra ABI flags are
needed; the `--host=$LFS_TGT` triplet (`i686-lfs-linux-gnu`) selects IA-32.

The dynamic linker path for x86 is:

```text
/lib/ld-linux.so.2
```

The LSB compatibility symlink is also created during installation:
```text
/lib/ld-lsb.so.3 → ld-linux.so.2
```

### Sanity Check

After building Glibc, verify the dynamic linker path:

```bash
echo 'int main(){}' | $LFS_TGT-gcc -xc -
readelf -l a.out | grep interpreter
```

Expected output:
```text
[Requesting program interpreter: /lib/ld-linux.so.2]
```

If building on an amd64 host, `readelf` will run on a 32-bit ELF; that is
expected and correct.

## Bootloader

GRUB BIOS is the standard bootloader for IA-32.  UEFI is rare on 32-bit x86
hardware; most real and virtual i686 machines use BIOS.

See [Bootloader Configuration — amd64 / x86 — GRUB BIOS](../part4-system-config/bootloaders.md#amd64--x86--grub-bios-or-efi).
